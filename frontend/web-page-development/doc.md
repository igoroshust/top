## БЭМ
БЭМ - это методология (не фреймворк) для именования CSS-классов, разработанная в Яндексе в 2005 году. Она помогает создавать масштабируемый, читаемый и поддерживаемый CSS-код, избегая конфликтов имён и каскадных проблем. БЭМ разделяет интерфейс на независимые компоненты, используя строгую систему нейминга. Основные принципы: независимость блоков, плоская структура и модифицируемость. БЭМ не зависит от препроцессоров (Sass/Less), но хорошо с ними сочетается.

БЭМ - компонентный подход к веб-разработке. В его основе лежит принцип разделения интерфейса на независимые блоки. Он позволяет легко и быстро разрабатывать интерфейсы любой сложности и повторно использовать существующий код, избегая copy-paste.

### Основные понятия
- Block. Независимый компонент интерфейса, который можно переиспользовать. Блок самодостаточен и не зависит от других блоков. Примеры: `header, button, menu`.
Название блока характеризует смысл (что это? - меню: menu; кнопка: button).
```html
<!-- Корректный пример. error - семантически осмысленный блок -->
<div class="error">
```
Блок не должен влиять на своё окружение, т.е. блоку не следует задавать внешнюю геометрию (в виде отступов, границ, влияющих на размеры) и позиционирование.
В CSS по БЭМ также не рекомендуется использовать селекторы по тегам или id. Таким образом обеспечивается независимость, при которой возможно повторное использование или перенос блоков с места на место.

**Принципы работы с блоками**
1. Вложенность
- Блоки можно вкладывать друг в друга
- Допустима любая вложенность блоков
```html
<header class="header">
    <div class="logo"></div>
    <form class="search-form"></form>
</header>
```


- Element. Часть блока, не имеющая смысла вне блока. Элемент всегда принадлежит блоку, например: `menu__item`, `menu__item--active`. **Название элемента характеризует смысл** (что это? - пункт: item, текст: text), а не состояние (какой, как выглядит).

Структура полного имени элемента соответствует схеме: `имя-блока__имя-элемента`. Имя элемента отделяется от имени блока двумя подчёркиваниями.

```html
<!-- Блок `search-form` -->
<form class="search-form">
    <!-- Элемент input блока search-form -->
     <input class="search-form__input">

    <!-- Элемент button блока search-form -->
    <button class="search-form__button">Найти</button>
</form>
```

#### Принципы работы с элементами (Вложенность, Принадлежность, Необязательность)
**Вложенность**
- Элементы можно вкладывать друг в друга;
- Допустима любая вложенность элементов;
- Элемент - всегда часть блока, а не другого элемента, то есть в названии нельзя использовать запись вроде `block__elem1__elem2`.
```html
<!-- Пример корректной структуры вложенных друг в друга элементов -->

<form class="search-form">
    <div class="search-form__content">
        <input class="search-form__input">
        <button class="search-form__button">Найти</button>
    </div>
</form>
```
Имя блока задаёт пространство имён, гарантирующее зависимость элемента от блока.

+ Блок может иметь вложенную структуру элементов в DOM-дереве:
```html
<div class="block">
    <div class="block__elem1">
        <div class="block__elem2">
            <div class="block__elem3"></div>
        </div>
    </div>
</div>
```
Однако эта же структура блока в методологии БЭМ всегда будет представлена плоским списком элементов:
```css
.block {}
.block__elem1 {}
.block__elem2 {}
.block__elem3 {}
```

Это позволяет изменять DOM-структуру блока без внесения правок в коде каждого отдельного элемента:
```html
<div class="block">
    <div class="block__elem1">
        <div class="block__elem2"></div>
    </div>

    <div class="block__elem3"></div>
</div>
```
Как итог: структура блока меняется, а правила для элементов и их названия остаются прежними.


**Принадлежность**
Элемент - всегда часть блока, и он не должен использоваться отдельно от него.
```html
<form class="search-form">
    <input class="search-form__input">
    <button class="search-form__button">Отправить</button>
</form>
```

**Необязательность**
Элемент - необязательный компонент блока. Не у всех блоков должны быть элементы.


#### Когда создавать блок, а когда - элемент?
Создание блока: элемент может быть использован повторно и не зависит от реализации других компонентов страницы.
Создание элемента: если фрагмент кода не может использоваться самостоятельно, без родительской сущности (блока). Исключение составляют элементы, реализация которых для упрощения разработки требует разделения на более мелкие части - подэлементы. В БЭМ-методологии нельзя создавать элементы элементов. В подобном случае вместо элемента необходимо создавать служебный блок.



- Modifier. Вариация блока или элемента, изменяющая его внешний вид или поведение. Модификатор добавляется к блоку/элементу для условных стилей. Примеры: `button--primary`, `menu__item--active`. Это сущность, определяющая внешний вид, состояние или поведение блока либо элемента. Особенности:
- Название модификатора характеризует внешний вид (какой размер? какая тема? размер: size_s, тема: theme_dark), состояние (чем отличается от прочих? отключён: disabled, фокусированный: focused) и поведение (как ведёт себя? как взаимодействует с пользователем? направление: directions_left-top)
- Имя модификатора отделяется от имени блока или элемента одним подчёркиванием (_). Именование с одним подчёркиванием - классическая схема, есть ещё вариант Two Dashes - с двумя тире (--). Выбор зависит от предпочтений команды.

#### Типы модификаторов

1. **Булевый**
- Используют, когда важно только наличие или отсутствие модификатора, а его значение несущественно. Например, отключён: disabled. Считается, что при наличии булевого модификатора у сущности его значение равно true.
- Структура полного модификатора соответствует схеме:
- `имя-блока_имя-модификатора`
- `имя-блока__имя-элемента_имя-модификатора`
```html
<form class="search-form search-form_focused">
    <input class="search-form__input">

    <!-- Элемент button имеет булевый модификатор `disabled` -->
     <button class="search-form__button search-form__button_disabled">
</form>
```

2. **Ключ-значение**
- Используют, когда важно значение модификатора (вроде меню с темой оформления islands: menu_theme_islands).
- Структура полного имени модификатора соответствует схеме: 
`имя-блока_имя-модификатора_значение-модификатора`
`имя-блока__имя-элемента_имя_модификатора_значение-модификатора`

```html
<form class="search-form search-form_theme_island">
    <input class="search-form__input">

    <!-- Элемент button имеет модификатор size со значением m-->
    <button class="search-form__button search-form__button_size_m">Нажать</button>
</form>
```
P.S. Невозможно использовать два одинаковых модификатора с разными значениями.


**Принципы работы с модификаторами**
Модификатор нельзя использовать самостоятельно. С точки зрения БЭМ, модификатор не может использоваться в отрыве от модифицируемого блока или элемента. Модификатор должен ИЗМЕНЯТЬ вид, поведение или состояние сущности, а не заменять её.

Пример:
```html
<form class="search-form search-form_theme_islands">
    ...
</form>
```

**МИКС**
Приём, позволяющий использовать разные БЭМ-сущности на одном DOM-узле. Миксы позволяют:
- Совмещать поведение и стили нескольких сущностей без дублирования кода;
- Создавать семантически новые компоненты интерфейса на основе имеющихся;

```html
<div class="header">
    <!-- К блоку `search-form` примиксован элемент `search-form` блока `header` -->
    <div class="search-form header__search-form"></div>
</div>
```
В данном примере совмещено поведение и стили блока search-form и элемента search-form блока header. Такой подход позволяет задать внешнюю геометрию и позиционирование в элементе header__search-form, а сам блок search-form оставить универсальным. Таким образом, блок можно использовать в любом другом окружении, потому что он не специфицирует никакие отступы. Это позволяет нам говорить о его независимости.













**Основная идея БЭМ**: интерфейс строится на наборе блоков, у каждого из которых есть свои элементы и модификаторы.

### Синтаксис
- Блок: `block-name` (кебаб-кейс или camelCase, но кебаб предпочтительнее);
- Элемент: `block-name__element-name` (двойное подчёркивание `__`);
- Модификатор: `block-name--modifier-name` или `block-name__element-name--modifier-name` (двойной дефис `--`);
- Составной модификатор: `block-name--modifier-name_value` (если значение не булево, например, `button--size_large`).

**Правила**
- Имена в lowercase, без пробелов
- Избегайте каскада (селекторы только по классам, без вложенности)
- Классы уникальны и описательны


### Примеры
1. **Блок с элементом**
```html
<!-- Блок --> 
<button class="button">
     <!-- Элемент -->
    <span class="button__text">Нажми меня</span>
</button>
```

2. **Блок с модификатором**
```html
<button class="button button--primary">Нажми меня</button>
<button class="button button--secondary">Отмена</button>

<!-- Модификатор button--primary изменяет цвет. Классы комбинируются (возможны несколько модификаторов) -->
```

3. **Блок с элементом и модификатором**
```html
<div class="menu">
    <ul class="menu__list">
        <!-- Пункты меню -->
        <li class="menu__item menu__item--active">Главная</li> <!-- Активный пункт -->
        <li class="menu__item">О нас</li>
    </ul>
</div>
```

4. **Пример на карточке товара**
```html
<div class="card card--featured"> <!-- card--featured модификатор для всей карточки -->
    <img class="card__image" src="product.jpg" alt="Товар">
    <h3 class="card__title">Название товара</h3>
    <p class="card__description">Описание.</p>
    <button class="card__button card__button--buy">Купить</button> <!-- card__button--buy модификатор для кнопки -->
</div>
```


5. **Блок + элемент + модификатор**
```html
<header class="header">
    <div class="header-paragraphs">
        <p class="header-paragraphs__item">...</p> <!-- Параграф как элемент блока -->
        <p class="header-paragraphs__item header-paragraphs__item--large">...</p> <!-- Крупный текст -->
    </div>
</header>
```






### Лучшие практики и преимущества
**Преимущества**
- Изоляция. Блоки независимы, нет конфликтов стилей.
- Переиспользование. Блоки можно копировать между проектами.
- Поддержка. Легко найти и изменить стили по имени класса.
- Производительность. Меньше каскада, быстрее рендеринг.
- Командная работа. Ясные имена для всех.

**Правила:**
- Не использовать каскад (`.block .element` - только `.block__element`).
- Модификаторы не переопределяют базовый стиль, а добавляют.
- Для миксов (несколько блоков на элементе): `<div class="block1 block2">` - допустимо, но с осторожностью.
- Избегайте глубоких вложенностей; держите плоскую структуру.


**Особенности**
В БЭМ элементы не должны нумероваться, так как это нарушает принцип переиспользуемости и семантики. Имена элементов должны отражать их назначение, а не порядковый номер. Нумерация может привести к следующим проблемам:
- Сложность поддержки: при изменении структуры (например, добавлении или удалении элементов) нумерация станет некорректной.
- Нарушение семантики: имя вроде `item-1` не передаёт смысл элемент, что затрудняет понимание кода.
- Ограничение переиспользования: элемент с номером теряет универсальность и становится привязан к конкретному месту в структуре.
